Search Engine Comparison
Engine Pros Cons
Bleve (Index)
  • Fastest response time
  • Complex text analysis (fuzzy, prefix)
  • Importance-based ranking
  • Supports huge datasets
  • Requires disk space for index
  • Needs pre-building step
PBF (Direct)
  • No pre-processing needed
  • Search raw OSM data directly
  • Zero disk overhead (uses .pbf)
  • Slowest (full scan)
  • Limited text search features
  • Not suitable for large regions
PMTiles (Direct)
  • No index building needed
  • Very fast spatial lookups
  • Cloud-native (supports HTTP range)
  • Limited text search (scanning layer)
  • Dependent on tile data schema
